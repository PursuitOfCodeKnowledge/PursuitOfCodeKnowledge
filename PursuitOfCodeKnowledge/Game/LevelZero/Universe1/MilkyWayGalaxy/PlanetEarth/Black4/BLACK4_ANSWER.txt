Чтобы выполнить задание:
Первое нужно посмотреть где ты находишься, используй команду pwd:
pwd
К примеру она тебе покажет путь /home/sergey/Desktop/PursuitOfCodeKnowledge.git/PursuitOfCodeKnowledge/Game/LevelZero/Universe1/MilkyWayGalaxy/PlanetEarth/Black4, получается ты сейчас находишься в Black4, 
тебе нужно вернуться назад используй команду cd ..:
cd ..
После этого набери команду ls или ls -a и посмотри что находится вокруг тебя какие файлы и директории:
ls
или
ls -a
Затем набери команду mkdir чтобы создать директорию Black (в том случае если ещё не создавал директорию Black):
mkdir Black
Затем перейди директорию в Black4:
cd Black4
Потом скопируй файл BLACK4.txt в директорию Black:
cp BLACK4.txt ../Black/
Затем проверь что скопировал файл BLACK4.txt в директорию Black при помощи команды ls:
ls ../Black/BLACK4.txt
После перенеси  файл файл BLACK4_ANSWER.txt в директорию Black:
mv BLACK4_ANSWER.txt ../Black/
Потом удали файл BLACK4.txt при помощи команды rm:
rm BLACK4.txt
или
rm -i BLACK4.txt
или
rm -f BLACK4.txt
После выйди из этой директории Black4:
cd ..
Затем удали директорию Black4:
rm -r Black4
После убедись что удалил директорию:
ls 
или
ls -a
Если исследовал уже директорию Black3, то установи пакеты postgresql и postgresql-contrib:
sudo apt-get update
sudo apt-get install postgresql postgresql-contrib
При установке пакетов postgresql и postgresql-contrib(если спросят) нажми клавишу y и нажми enter
После установке пакетов, скачиваем базу данных например demo при помощи команды wget:
wget https://edu.postgrespro.ru/demo-big.zip
Когда скачается архив, вводим команду ls:
ls
И мы должны увидить архив demo-big.zip, распаковываем этот архив при помощи 7z:
7z x demo-big.zip
Затем вводим команду ls:
ls
И вы должны увидить файл с расширением sql (demo-big-20170815.sql)
После этого удаляем архив demo-big.zip при помощи команды:
rm demo-big.zip
Затем заходим в postgres при помощи команды:
sudo -u postgres psql postgres
Потом задаем пароль для пользователя postgres набирая команду в терминале:
\password postgres
вводим пароль, незабываем пароль при вводе не отображается и нажимаем enter,
затем ещё раз повторно вводим пароль и нажимаем enter.
После смены пароля, набираем команду чтобы просмотреть путь где находится конфигурационный файл pg_hba.conf:
SHOW hba_file;
Он покажет путь например такой:  /etc/postgresql/15/main/pg_hba.conf
Выделяем его и копируем этот путь, после этого выходим из postgres при помощи команды \q:
\q
Затем вводим команду для редактирования конфигурационного файла при помощи команды nano:
sudo nano /etc/postgresql/15/main/pg_hba.conf
После открытия редактора, прокрутите текст вниз при помощи стрелочек и найдите строку:
local   all             all                                      peer

Замените её на (если в правом столбце уже было trust или md5, то не меняйте настройку):
local   all             all                                      md5

Теперь найдите вот эту строку:
local   all             postgres                                      peer

Замените её на (если в правом столбце уже было trust или md5, то не меняйте настройку):
local   all             postgres                                md5

Теперь найдите вот эту строку:
local   replication     all                                     peer

Замените её на (если в правом столбце уже было trust или md5, то не меняйте настройку):
local   replication     all                                     md5

Теперь нажимаем клавиши чтобы сохранить ctrl+o, затем нажимаем enter и выходим из файла при помощи ctrl+x
После идём к другому конфигурационному файлу и редактируем его:
sudo nano /etc/postgresql/15/main/postgresql.conf

После открытия редактора, прокрутите текст вниз при помощи стрелочек и найдите строку и расскоментируйте её убрав знак # перед listen_addresses:
#listen_addresses = 'localhost' 

Замените её на:
listen_addresses = 'localhost' 
или
listen_addresses = '*'

После этого нажмите сохранить при помощи клавиш ctrl+x, затем нажать клавишу enter и выйти из файла при помощи клавиш ctrl+x
Теперь перезагружаем службу:
sudo systemctl restart postgresql

Затем можно установить базу данных demo при помощи команды:
psql -f demo-big-20170815.sql -U postgres

Когда установилась база данных demo, удаляем файл с расширением sql (demo-big-20170815.sql):
rm demo-big-20170815.sql 

После этого заходим в postgres:
sudo -u postgres psql postgres

Вводим пароль от пользователя postgres, который мы создавали до этого

Теперь посмотрим какие базы данных при помощи команды \l:
\l

Видим demo и переходим в неё при помощи команды \c:
\c demo

Затем можно посмотреть список пользователей баз данных:
\du

Потом можно посмотреть какие таблицы в базе данных:
\dt

Затем можно посмотреть подробнее о любой таблице, например bookings:
\d bookings

Потом выполняем выборку по запросу:
SELECT * FROM bookings LIMIT 5;

После этого выходим из базы данных demo:
\q

Потом дальше иследуй другую директорию, например Black5
cd Black5

When installing the postgresql and postgresql-contrib packages(if you will ask you), press the y key and press enter
After installing the packages, download the database for example demo using the command wget:
wget https://edu.postgrespro.ru/demo-big.zip

ls

7z x demo-big.zip

\password postgres
enter the password, forget the password is not displayed when entering, and press enter,

After changing the password, type the command to view the path where the pg_hba.conf configuration file is located:
SHOW hba_file;
It will show a path like this: /etc/postgresql/15/main/pg_hba.conf
Select it and copy this path, then exit postgres using the \q:
\q
Then enter the command to edit the configuration file using the nano command:
sudo nano /etc/postgresql/15/main/pg_hba.conf
After opening the editor, scroll down using the arrows and find the line:
local   all             all                                      peer

Replace it with (if trust or md5 was already in the right column, then do not change the setting):
local			all		 all		 md5

Now find this line:
local   all             postgres                                      peer

Replace it with (if trust or md5 was already in the right column, then do not change the setting):
local		 all		 postgres		 md5

Now find this line:
local			 replication	    all                       peer

Replace it with (if trust or md5 was already in the right column, then do not change the setting):
local		 replication		 all			 md5

Now press the keys to save ctrl+o, then press enter and exit the file using ctrl+x
Then we go to another configuration file and edit it:
sudo nano /etc/postgresql/15/main/postgresql.conf

After opening the editor, scroll down using the arrows and find the line and comment it by removing the # sign in front of listen_addresses:
#listen_addresses = 'localhost' 

Replace it with:
listen_addresses = 'localhost'
or
listen_addresses = '*'

After that, press save using ctrl+o, then press enter and exit the file using ctrl+x.

sudo systemctl restart postgresql

psql -f demo-big-20170815.sql -U postgres

rm demo-big-20170815.sql

sudo -u postgres psql postgres
